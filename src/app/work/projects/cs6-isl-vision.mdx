## The Problem

Indian Sign Language has 262 distinct signs. The deaf and hard-of-hearing community in India — estimated at over 18 million people — relies on ISL as a primary mode of communication. Yet ISL recognition tooling is nearly non-existent as open-source software, and the models that do exist are built on CNN-LSTM architectures that struggle with the core challenge: **signs are temporal**. A gesture isn't a frame — it's a sequence of frames with motion dependencies across time.

Existing CNN-LSTM approaches were hitting ~66% Top-1 accuracy on 262-class classification. That's not good enough to be useful in practice.

---

## The Hypothesis

Vision Transformers trained on video (VideoMAE) learn temporal attention across frames by design — they don't need hand-crafted temporal feature engineering. If fine-tuned on ISL data, they should handle the motion complexity that CNN-LSTMs approximate poorly.

The question was whether transfer learning from a large video dataset (Kinetics-400) would translate to a domain as specialized as Indian Sign Language.

---

## What I Built

**Model:** Fine-tuned `VideoMAE` (ViT-B/16 pretrained on Kinetics-400) on a 262-class ISL video dataset using HuggingFace Transformers and PyTorch on Kaggle GPU (T4 x2).

**Real-time demo:** A desktop application that:
1. Records a **1.5-second gesture** via webcam
2. Extracts frames and runs VideoMAE inference in a **background thread** (keeps UI responsive)
3. Displays the **predicted sign + confidence score** in real time on CPU hardware

## Why VideoMAE Over CNN-LSTM

CNN-LSTMs process spatial features (CNN) and then model temporal dependencies (LSTM) as a two-stage pipeline. The temporal signal is a downstream artifact of spatial features — which means motion across frames is never directly attended to.

VideoMAE uses **masked video modeling** during pre-training: it masks ~90% of video patches and learns to reconstruct them, forcing the model to build rich temporal representations. During fine-tuning on ISL, this translates directly — the model already understands how motion unfolds across a short video clip.

The 25-point accuracy jump wasn't primarily about architecture size. It was about the right inductive bias for the problem.

## What I'd Do Next

The current model runs on CPU with a 1.5-second clip window. Three things I'd improve with more time:

**Reduce clip window to 0.5s** — 1.5 seconds is too long for natural real-time interaction. Faster inference would require either model distillation or a smaller ViT variant (VideoMAE-Small).

**Continuous recognition** — the current flow is gesture-by-gesture. A sentence-level system would require sliding window inference and a language model layer to assemble signs into coherent output.

**Dataset expansion** — 262 classes covers core ISL vocabulary but not full conversational range. Collecting or sourcing data for 500+ classes would meaningfully improve utility.
